## 二、期望实现的功能

### 2.1 系统基本功能

| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 用户注册登录 | 卖家/买家注册,后台管理员预设账号登录 |
| F-02 | 四角色权限 | 卖家/买家/拍卖管理员/系统管理员,各有标准操作 |
| F-03 | 拍品 CRUD | 卖家创建/编辑/删除拍品,含图片上传(单次最多 5 张) |
| F-04 | 拍品审核 | 拍卖管理员审核拍品(通过/驳回),驳回可附原因 |
| F-05 | 拍卖场次 | 卖家对审核通过拍品开启拍卖,设定结束时间 |
| F-06 | 定时自动开拍 | 到达开始时间自动将 scheduled→live |
| F-07 | 实时竞价 | WebSocket 订阅 + HTTP 双重出价通道,全播广播 |
| F-08 | 定时结拍 | 到达结束时间自动结算中标者,生成订单 |
| F-09 | 订单支付 | 买家模拟支付,状态流转: pending→paid |
| F-10 | 卖家发货 | 卖家标记已发货: paid→shipped |
| F-11 | 确认收货 | 买家确认: shipped→completed |
| F-12 | 订单取消 | 买家/管理员可取消未支付或已支付订单 |
| F-13 | 争议处理 | 买家/卖家发起争议 → 拍卖管理员受理/解决/驳回 |
| F-14 | 用户管理 | 系统管理员 CRUD 用户,启用/禁用 |
| F-15 | 系统配置 | 系统管理员读写配置项 |
| F-16 | 操作日志 | 自动记录 9 类关键操作,支持审计查询 |
| F-17 | 数据看板 | 后台首页 6 个统计卡片 |

## 三、技术栈

| 层 | 技术 |
|------|------|
| 移动端 | 微信小程序原生 (WXML/WXSS/JS) |
| 后台前端 | Vue3 + Vite + Element Plus + Pinia + vue-router |
| 后端 | Node.js + Express + TypeScript |
| 数据库 ORM | Prisma 7 + @prisma/adapter-pg |
| 业务库 | PostgreSQL 15 |
| 向量库 | pgvector 0.8.4 扩展 (同一 PG 实例) |
| 缓存 | Redis 7 (竞价排行/倒计时/在线) |
| 实时通信 | WebSocket (ws 库) |
| 大模型 Chat | DeepSeek V4 Flash (opencode.ai) |
| 大模型 Embedding | 硅基流动 BAAI/bge-m3 (1024 维) |
| 联网搜索 | Tavily Search API |
| 鉴权 | JWT (jsonwebtoken) |
| 密码 | bcryptjs |
| 参数校验 | zod |
| 容器化 | Docker Compose |

## 四、遇到的 Bug 与解决方案

### 4.1 已解决的 Bug

| 编号 | Bug 描述 | 原因 | 解决方案 |
|------|----------|------|----------|
| B-01 | ESPERM 安装依赖失败 `esbuild` | Windows 上 esbuild.exe 被 Defender 锁定,pnpm rename 操作失败 | 改用 npm 安装,配置 `.npmrc` 用 hoisted 模式 |
| B-02 | Prisma 7 不支持 `datasource.url` | Prisma 7 去掉了 schema 中 datasource url,需要 driver adapter | 加 `prisma.config.ts` 配置,用 `@prisma/adapter-pg` |
| B-03 | PrismaClient 需传 adapter 构造 | Prisma 7 engine type=client 强制要求 adapter 或 accelerateUrl | 改用 `PrismaPg` adapter + `Pool` |
| B-04 | pgvector `registerTypes` 不可用 | pgvector npm 包 API 与文档不一致 | 改用原生 SQL 传 `$x::vector` 显式转换 |
| B-05 | 微信小程序 `wx.request` 400 JSON 解析失败 | `wx.request` 默认 form-encoded,Express 只解析 JSON | 用封装好的 `utils/api.js` 设置 `Content-Type: application/json` |
| B-06 | Windows 上 curl.exe JSON 转义问题 | PowerShell 中 `'{\\"key\\":\\"val\\"}'` 被额外转义 | 改用 `WriteAllText` 写入文件 + `--data-binary @file` |
| B-07 | `DataCloneError` XMLHttpRequest JSON | 请求体包含非 UTF-8 字符 | 用 `System.Text.UTF8Encoding($false)` 无 BOM 写入 |
| B-08 | 拍卖场次"请求失败(400)" | 拍品已存在拍卖场次,按钮仍显示"开启拍卖" | `listMine`加 `include: { auction }`,前端判断 `item.auction` 是否存在 |
| B-09 | 订单 403"非本人订单" | 卖家登录后在"我的订单"中看到买家订单并点支付 | 添加 `myUserId` 比对,仅买家显示支付按钮 |
| B-10 | 微信开发者工具 400 "用户名已被占用" | 输入框中文字符编码错误 | 改用 `utils/api.js` 封装的方法(post),确保 Content-Type |
| B-11 | AI 文案生成后 textarea UI 错乱 | 微信原生 textarea 溢出覆盖下方按钮 | `auto-height` + `max-height: 400rpx` + `overflow-y: auto` |
| B-12 | `selectedColor` 缺字符 | app.json 中 `"#1296db"` 写成了 `"#296db"` | 修正为 `"#1296db"` |
| B-13 | 小程序 `require` 模块错误时白屏 | `api.js` 使用了 ES6 语法(默认参数/箭头函数/解构)但增强编译未正确处理 | 全部改写为 ES5 兼容语法(`var`/`function`) |
| B-14 | 估价助手看不到用户消息 | `scroll-view` 需要固定高度,flex:1 在模拟器中不工作 | 换成普通 `view` + `overflow-y: auto` + flex 布局 |
| B-15 | Prisma 7 迁移 `SoldItemEmbedding` 无法识别 `Unsupported("vector(1024)")` | Prisma migrate 会建立表,但 `Unsupported` 字段类型仅 SQL 中手动定义 | migrate 自动创建表结构,vector 类型由 pgvector 镜像自带 |
| B-21 | `GET /api/bids/auction/:auctionId` 需登录 | `index.ts` 将 `/api/bids` 整组挂载了 `auth` 中间件 | 移除路由级 `auth`,各子路由各自控制 |
| B-22 | `DELETE /items/:id` 无拍卖检查 | 拍卖进行中的拍品可被软删除,导致场次悬空 | 新增 `auction.findFirst` 查询 `scheduled/live` 状态,存在则拒绝删除 |
| B-23 | `forceEnd()` 未写入成交向量 | `SoldItemEmbedding` 仅在 `pay()` 中索引 | `forceEnd()` 交易后调用 `indexSoldItem()` 写入 |
| B-24 | `endExpiredAuctions()` 未写入成交向量 | 同上,自动结拍不触发向量索引 | 创建订单后调用 `indexSoldItem()` |
| B-25 | `POST /admin/auctions/:id/force-cancel` 缺少 `reason` 校验信息 | 无单独文档,测试时直接返回 422 | 已在 admin 路由中明确 `reasonSchema` (zod: min1 max200) |
| B-26 | Docker Desktop daemon.json 配置错误导致启动失败 | `registryMirrors` 使用了驼峰命名,应使用连字符 `registry-mirrors` | 修正为 `registry-mirrors`,重启 Docker Desktop |
| B-27 | 首页显示已结束/已成交的拍卖 | `index.js` 调用 `/auctions` 无状态过滤 | 前端 `index.js` 中添加 `filter` 只显示 `live` 和 `scheduled` |
| B-28 | 订单列表不显示拍卖场次 ID | `orders.wxml` 只显示拍品名称和成交价 | 添加 `item.auctionId` 显示"拍卖场次 #ID" |
| B-29 | 争议页面拍卖场次 ID 输入框无法输入 | `<input disabled="{{auctionId !== null}}"` 且无 `bindinput` | 移除 `disabled`,添加 `bindinput="onAuctionInput"` 事件处理 |
| B-30 | 登录页"先注册再登录"导致误注册新账号 | 小程序登录逻辑先尝试注册,失败再登录 | 重写 `login.js`,分离登录/注册为独立页面,新建 `pages/register/` |
| B-31 | 争议原因提示语"请填写详细原因"不准确 | 字数不足5字但非空的提示语误导 | 改为"字数不达标" |
| B-32 | AI 估价助手回复显示空白灰色方块 | AI 返回 markdown 格式(`**加粗**`、`|表格|`)在小程序 `<view>` 中不渲染 | 添加 `stripMarkdown()` 函数移除 markdown 标记,改用 `<text>` 组件 |
| B-33 | AI 回复被截断不完整 | `chatCompletion` 默认 `maxTokens=1024` 过小 | 改为 `maxTokens=4096` |
| B-34 | `valuation.js` 中 `_sessionId` 未在 `data` 中声明 | 使用 `this.data._sessionId` 但未在 data 中初始化 | 在 `data` 中添加 `sessionId: 0`,改用 `this.data.sessionId` |
| B-35 | WXML 中 `wx:for` 嵌套变量名冲突 | 外循环默认变量 `item` 被内循环覆盖 | 统一使用 `item` 作为默认变量,或显式指定 `wx:for-item` |
| B-36 | 估价助手气泡 CSS `overflow: hidden` 隐藏内容 | `.line` 类中 `overflow: hidden` 导致长文本溢出隐藏 | 移除 `overflow: hidden`,改用 `word-break: break-word` |
| B-37 | 发布拍品后返回发布页表单未清空 | tabBar 页面切换不会重置数据 | 添加 `onShow()` 重置表单和图片列表 |
| B-38 | 控制台"累计成交"始终为0 | 只统计 `completed` 状态订单,但成交订单初始为 `pending` | 改为统计 `paid`/`shipped`/`completed`(后按用户要求改回仅 `completed`) |
| B-39 | 买家订单确认收货按钮不显示/卖家无发货按钮 | 页面用 `buyerId` 比对判断角色,但未获取当前用户角色 | 添加 `myRole` 数据字段,用角色判断显示发货/确认收货按钮 |
| B-40 | HTTP 出价广播缺少 `currentPrice` 字段 | HTTP 路由 `broadcastAuction` 未传入 `currentPrice` | 补上 `currentPrice: currentPrice` 字段 |
| B-41 | 拍卖详情页实时竞拍不更新 | 页面仅 `onLoad` 加载一次数据,无 WebSocket 监听 | 添加 WebSocket 连接,收到 `bid` 消息时更新当前价 |
| B-42 | 首页/订单/我的拍品/竞拍页不显示拍品图片 | WXML 中未添加 `<image>` 组件 | 各页面添加图片显示,`imgBase` 从 `apiBase` 拼接 |
| B-43 | 未登录点击"参与竞拍"/"发起争议"直接跳转登录页 | 直接 `wx.navigateTo` 无提示 | 改为在本页显示未登录提示和"前往登录"按钮 |
| B-44 | 管理员禁用用户后仍可登录 | 登录接口和 `auth` 中间件未检查 `user.status` | 登录时检查 `status===0` 拒绝登录,auth中间件每请求检查状态 |

## 九、技术要点记录

### 9.1 AI 描述优化流程
```
卖家输入[名称+成色+原价] → POST /api/ai/optimize-description
 → DeepSeek V4 Flash → 生成 200-400 字文案
 → 存入 AiDescriptionHistory 表 → 返回给前端 textarea
 → 卖家可编辑后提交拍品
```

### 9.2 RAG 估价助手流程
```
① 历史成交向量化(3个触发时机):
  买家支付订单 → order.pay()  → indexSoldItem()
  管理员强制结拍 → forceEnd() → indexSoldItem()
  定时器自动结拍 → endExpiredAuctions() → indexSoldItem()
  → bge-m3(硅基流动) → vector(1024) → INSERT INTO SoldItemEmbedding

② 估价查询(两步):
  第一步 POST /api/ai/valuation-search (联网搜索):
    卖家提问 → Tavily Search API → 返回 webContext
  第二步 POST /api/ai/valuation (AI 生成):
    bge-m3 向量化查询 → pgvector cosine 检索(<=>) → 同类 TOP5
    → 拼接联网搜索结果到 prompt → DeepSeek 生成估价建议
    → SSE 流式输出 → sessionId 支持多轮追问
```

### 9.3 AI 流式 SSE 输出
```
服务端: POST /api/ai/valuation-stream
  → Content-Type: text/event-stream
  → OpenAI client 设置 stream: true
  → 逐 chunk 写入 res.write() SSE 格式

小程序端: wx.request({ enableChunked: true })
  → onChunkReceived 回调接收 ArrayBuffer
  → TextDecoder('utf-8', {stream:true}) 解码
  → 逐字符追加到 streamingText 实时渲染
```

### 9.4 WebSocket 竞价
```
客户端连接: ws://localhost:3000/ws?token=<jwt>
订阅: {"action":"subscribe","auctionId":1}
出价: {"action":"bid","amount":100}
服务端: bidService.placeBid() → 落库 → broadcastAll(在场所有客户端)

WebSocket 消息处理:
  subscribe → 加入 auctionSubscribers Map
  bid → placeBid → 成功则广播 bid,失败则返回 error
  前端新增 msg.action==='error' 处理,弹窗显示错误
```

### 9.5 定时调度
```
Scheduler 每 10 秒:
  ① startScheduledAuctions(): scheduled + startTime<=now → live
  ② endExpiredAuctions():
     live + endTime<=now
       → 有出价: settled + 创建order + indexSoldItem()写入向量
       → 无出价: ended
```

### 9.6 联网搜索 (Tavily)
```
POST https://api.tavily.com/search
  body: { api_key, query, max_results:5, include_answer:true }
  → 返回 answer(摘要) + results[](标题+内容+URL)
  → 拼接为 webContext 注入 Prompt
  → 无 Tavily Key 时静默跳过,不影响主流程
```
